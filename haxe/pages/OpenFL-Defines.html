OpenFL Defines, And Their Meanings by Shahar Marcus


Over Time, because of OpenFL's growth and optimizations, it introduced specific defines, which enable/disable certain features.

If you don't know what defines are, visit this link
Define Name Meaning Versions 
<haxedef name="queue_experimental_optimization"/>
 Enables a new and optimized game loop - boosts performance, and takes less memory 9.2.0+ 
<haxedef name="openfl_dynamic"/>
 Allows dynamic property access on three main OpenFL classes - MovieClip, DisplayObject and Error 5.0.0+
Haxe 3 Only		 
<haxedef name="openfl_debug"/>
 Prevents signing the program with the "release" tag, allows debugging information from being generated for openfl classes 
<haxedef name="openfl_trace_text_layout_groups"/>
 Debugging option, traces positional information about text layout groups, and the text they contains. 4.9.0+ 
<haxedef name="openfl_strict"/>
 Disables any unsafe/unimplemented/unstable features that are currently offered 8.9.1+ 
<haxedef name="openfl_disable_cacheasbitmap"/>
 Disables object.cacheAsBitmap's functionality 9.1.0+ 
<haxedef name="openfl_disable_filters"/>
 Disables visual filters entirely - blocks them from rendering 9.1.0+ 
<haxedef name="gl_stats"/>
 Adds additional debugging information to the FPS counter's text: total draw calls, stage draw calls this second, stage3d draw calls this second 6.5.3+ 
<haxedef name="openfl_disable_handle_error"/>
 Disables the UncaughtErrorEvent.UNCAUGHT_ERROR event 
<haxedef name="openfl_pool_events"/>
 Uses a pool to store events. when used, reduces the amount of memory used when lots of listeners are added 8.9.0 - 9.0.2 
<haxedef name="openfl_always_dispatch_mouse_events"/>
 Instead of waiting for the mouse event to bubble up, OpenFL dispatches that event instantly 8.9.0+ 
<haxedef name="openfl_disable_display_render"/>
 Disables rendering of objects entirely 
<haxedef name="openfl_always_render"/>
 renders every single object to the screen, even if it isn't meant/set to not be rendered 
<haxedef name="openfl_force_sw_graphics"/>
<haxedef name="force_sw_graphics"/>
 Renders everything with software rendering. You need to use only one of these defines. 8.0.0+ 
<haxedef name="openfl_force_hw_graphics"/>
<haxedef name="force_hw_graphics"/>
 Renders everything with hardware accelerated rendering. You need to use only one of these defines. 8.0.0+ 
<haxedef name="openfl_power_of_two"/>
 Makes sure the generated opengl texture for images have powers of two width/height (32, 64, 128, 256, etc.). Only needed for older devices which don't support textures with arbitrary dimensions. It's a pretty CPU-intensive process (creates another copy of the image, but with the required size), and it is recommended to use textures with powers of 2 width/height this define is used anyway to prevent stutters. 3.3.3+ 
<haxedef name="openfl_disable_graphics_upscaling"/>
 Disables scaleX and scaleY - when enabled, they're always set to 1. 9.0.2+ 
<haxedef name="openfl_disable_context_cache"/>
 Disables the use of cache in the Context3D class. 8.5.0+
