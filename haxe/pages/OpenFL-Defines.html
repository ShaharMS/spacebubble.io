OpenFL Defines, And Their Meanings by Shahar Marcus


Over Time, because of OpenFL's growth and optimizations, it introduced specific defines, which enable/disable certain features.

If you don't know what defines are, visit this link
Define Name Meaning Versions 
<haxedef name="queue_experimental_optimization"/>
 Enables a new and optimized game loop - boosts performance, and takes less memory 9.2.0+ 
<haxedef name="openfl_dynamic"/>
 Allows dynamic property access on three main OpenFL classes - MovieClip, DisplayObject and Error 5.0.0+
Haxe 3 Only		 
<haxedef name="openfl_debug"/>
 Prevents signing the program with the "release" tag, allows debugging information from being generated for openfl classes 
<haxedef name="openfl_trace_text_layout_groups"/>
 Debugging option, traces positional information about text layout groups, and the text they contains. 4.9.0+ 
<haxedef name="openfl_strict"/>
 Disables any unsafe/unimplemented/unstable features that are currently offered 8.9.1+
