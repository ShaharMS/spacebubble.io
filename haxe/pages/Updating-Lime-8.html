Struggling with Updating to Lime 8? by Shahar Marcus


Well, you came to the right place!


The new version of Lime 8 is a lot more powerful than the previous one, using more up to date libraries. But - theres a catch - Lime 8 gets its libraries straight from git, and that means you need to do a couple things for it to work


1. Verify You Have Lime 8


You can check this by typing haxelib list in your terminal.
Then, you should get something like this:

After getting that list, you'll need to check if Lime 8/git is in there. (In my case, im using lime from git, which is lime 8)

if lime isn't updated, you can either update from haxelib, with

haxelib update lime

or, pull the update from git:

haxelib git lime https://github.com/openfl/lime

If the wanted version (8 or git) isn't surrounded by [brackets] set the version:

haxelib set lime 8

or:

haxelib set lime git


If you updated your lime version, restart your code editor to let haxe update the libraries to use.


2. Pull The Submodules

Lime 8 has replaced its forks with git submodules, which means we need to pull those submodules to update the libraries inside them.

How Should I do that?

Actually, quite simple:
  1. Go to the Lime's containing folder - for example, in my computer its haxelibs/lime
  2. Open a terminal inside that folder, or a terminal that does actions inside that folder
  3. Type git submodule update --init
  4. Press enter, and wait a bit :)


3. Rebuild Lime's Tools And Binaries

Now, after making sure lime and its dependencies are up to date, we can tell lime to rebuild its libraries. that would ensure no core features are missing the next time you compile your application.
First, rebuild the lime tools themself, with

lime rebuild tools

Second, rebuild your platform-specific binaries:

lime rebuild windows|mac|linux...

Third, rebuild your target-specific binaries:

lime rebuild hl|js...


Now, the next step, is to compile, and enjoy ;)
